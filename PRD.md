# Vibe Wall - PRD

## 1. 一句话描述

一个极其简单的「情绪句子墙」，用户登录后可以发一条当下的 vibe 句子，并在公共墙上看到所有人的 vibe。

## 2. 目标用户

- 经常刷 X / 小红书 / Telegram channel 的互联网打工人。
- 想要一个轻量、不严肃的「情绪出口」，又不想经营完整社交账号的人。

## 3. 核心用户故事

1. 作为一个用户，我可以用 Google 一键登录，而不用再注册一个新账号。
2. 作为一个用户，我可以快速发一条 vibe 句子并打一个简单标签，比如「摆烂」「卷」「emo」。
3. 作为一个用户，我可以在一个页面看到所有人的公开 vibe 句子，按时间倒序。
4. 作为一个用户，我可以删掉自己发的 vibe 句子。

## 4. 功能范围（V1）

### 必做

- Firebase Auth + Google 登录 / 登出。
- 文本输入框 + 标签选择（下拉 / chips）。
- 提交后写入 Firestore。
- 页面加载时从 Firestore 读取最近 50 条 vibe。
- 只允许作者自己删除自己的 vibe。
- 部署到 Vercel，保证基本的环境变量配置。

### 不做（未来可能）

- 点赞 / 评论。
- 多语言 / i18n。
- 高级过滤（按标签筛选、搜索）。
- 复杂的用户主页。

## 5. 关键指标（玩票级）

- 首屏加载时间 < 2s（Vercel + Next.js 默认基本能满足）。
- 未登录用户看到 CTA 登录按钮，并能 3 秒内完成登录跳转。
- 发一条 vibe 的交互在 3 步以内：登录 → 输入 → 提交。

## 6. 非功能性要求

- 初始版本只适配移动端 + 小屏桌面（Tailwind 响应式）。
- 代码结构遵守 APPFLOW / BACKENDSTRUCTURE 中定义的目录。
- 任何秘密配置（Firebase keys 等）必须走环境变量，不写死进 repo。
